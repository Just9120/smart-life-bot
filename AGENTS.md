## Working rules

- Do not expand scope without explicit request.
- Do not modify unrelated parts of the project.
- Update documentation together with any architecture, config, workflow, deployment, or behavior change.
- Prefer simple solutions without quality loss.
- Never add secrets, private keys, or real tokens.
- If something is not yet defined, leave an explicit placeholder and mark it as pending.
